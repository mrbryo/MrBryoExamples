# MrBryoExamples
**A Sandbox for World of Warcraft Addon Feature Experimentation**

---

## ✨ Overview

`MrBryoExamples` is designed as a place to view examples of code and UI objects for World of Warcraft addon developers and advanced users. This addon does not perform a specific core function; instead, it serves as a collection of UI elements, modules, and interactive components intended to demonstrate or allow quick testing of various WoW API features.

If you are learning about dropdown menus, complex event handling, modular scripting, or integrating different aspects of the WoW API, this addon provides pre-built frameworks ready for use.

Note: Much of the code is a copy and paste from ![Better Addons](https://www.better-addons.com) with some changes I've made for my own purposes. I will likely use this as my starter template...eventually. I will try to mark what was copied from their site.

> 💡 **Disclaimer:** Because this is an example and testing addon, some components might be unstable, incomplete, or designed purely to demonstrate a feature rather than provide production-ready code. Always treat it as educational material!

## ⚙️ Features Showcase (Examples)

While the content inside `MrBryoExamples` changes based on what is being tested, core features you will find built into this sandbox include:

*   **Interactive Dropdown Menus:** Multiple implementations to test different behaviors of dropdown widgets.
*   **UI Layout Testing:** Examples showing how to structure and place complex modules within the WoW UI.
*   **Event Handling:** Snippets showing standard event handling setup/configuration.
*   **Modular Components:** Self-contained functional blocks that can be easily copied or adapted for new projects.

## 📥 Installation

1.  Locate your World of Warcraft `Interface\AddOns` folder.
2.  Place the entire `MrBryoExamples` folder inside the `AddOns` directory.
3.  Launch WoW and ensure the addon is enabled in the UI.

> **Note:** If you are contributing to this project, please consult the main codebase for specific file structures and best practices!

Alternatively, you can use Curse's application to install it.

## 🧑‍💻 Development & Usage

### For Developers:
This addon is structured to be highly modular. When experimenting with a new feature (e.g., changing how dropdown options load or how a button callback functions), we recommend the following workflow:

1.  Identify the specific component you wish to test.
2.  Copy that component's relevant script files into your own dedicated project.
3.  Adjust constants and functionality as needed for your use case.

### For Users:
If you are simply trying things out, feel free to click on or interact with the modules present in your UI frame(s) and see what happens! This is a sandbox—don't worry about breaking anything; that's the point!
